# 🛡️ Coding Safety Rules

에이전트가 코드를 수정할 때 발생할 수 있는 데이터 유실 및 런타임 에러를 방지하는 규칙입니다.

## 1. UI & 로직 보호
- **기존 로직 보존**: UI 디자인 변경 시 기존 이벤트 핸들러나 API 연동 로직이 삭제되지 않도록 1순위로 보호합니다.
- **전체 구조 파악**: 부분 수정 시에도 파일 전체를 정독(혹은 Outline 파악)하여 심볼 간 관계를 명확히 합니다.

## 2. 치명적 사고 방지
- **심볼 체크**: 존재하지 않는 변수나 메서드 참조를 금지합니다.
- **데드 코드 금지**: 리팩토링 후 우회된 기존 코드를 남겨두지 않습니다.
- **빌드 검증**: 수정 후 `npm run build` 등을 통해 런타임 에러를 사전에 차단합니다.

## 3. 시각적 검증 (Visual Verification)
- **실시간 확인**: 프론트엔드 코드 수정 직후에는 반드시 브라우저 도구(`read_browser_page` 등)를 사용하여 변경 사항이 의도대로 반영되었는지 직접 확인합니다.
- **최종 URL 보고**: 작업이 완벽히 확인된 경우에만 사용자에게 완료 보고를 하며, 이때 반드시 활성화된 URL을 제공하고, 가능한 경우 페이지를 직접 열어서 보여줍니다.

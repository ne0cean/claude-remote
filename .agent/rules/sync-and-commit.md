# 🔄 Sync & Commit Rules

멀티 PC/에이전트 환경에서 작업 연속성을 보장하기 위한 동기화 및 커밋 규칙입니다.

## 1. 커밋 메시지 강제 포맷
모든 커밋은 세션 종료 시 다음 형식을 따릅니다:
```text
[타입]: 작업 요약
NOW: 완료한 구체적 내용
NEXT: 다음 개발자가 이어받을 작업
BLOCK: 미해결 이슈
FILE: 수정된 주요 파일
```

## 2. 자동 저장 (Auto-Sync) 지침
- `scripts/auto-sync.sh`가 백그라운드에서 생성하는 커밋(`sync:` 프리픽스)을 유효한 작업 맥락으로 인정합니다.
- **선제적 최적화**: 자동 저장 전, 작업 중단 시간이 길면 `@.agent/workflows/self-optimization.md`를 호출하여 구조적 결함을 자율적으로 수정합니다.
- 세션 재개 시 자동 저장된 로그를 읽고 마지막 상태를 즉시 파악합니다.

## 3. URL 가시성
- 포트 변경이나 배포 URL 생성 시 즉시 보고하고 `.context/CURRENT.md`에 기록합니다.

## 4. 자율적 사용량 감지 (Self-Sensing)
- **맥락 부하 감지**: 대화가 30턴 이상 지속되거나, 응답 속도가 현저히 느려지면 모델 한도 도달을 스스로 예측합니다.
- **선제적 제안**: 한도 도달 전 사용자에게 `./scripts/model-exit.sh` 실행을 통한 모델 전환을 제안합니다.
- **상시 대기**: `auto-sync.sh`가 생성하는 `.context/HANDOFF_PROMPT.txt`가 항상 최신인지 확인합니다.
